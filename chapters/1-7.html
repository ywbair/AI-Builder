一、大模型时代的核心跃迁

突破一：统一的海量理解能力

一个模型能同时理解文字、代码、图像，不再是各自独立的"专家系统"。

突破二：涌现能力（Emergent Abilities）

当模型规模超过某个临界点，突然涌现出小模型没有的能力——复杂推理、代码生成、多步骤规划。量变引起质变。

突破三：自然语言作为通用接口

人类用自然语言描述需求，AI理解并执行。第一次，机器真正听懂了人类的意图。

技术里程碑

对话界面引爆行业，GPT-4成为首个图灵测试级别的大模型
2023-2024年 开源崛起

Transformer 核心原理：注意力机制

Transformer 通过"注意力机制"理解词与词之间的关系。比如这句话：

"我把苹果手机充了电，现在可以用了"

LLM需要理解"苹果"在这里是"手机"而不是"水果"，靠的就是词与词之间的"注意力"权重。

三、Token：LLM的计量单位

什么是 Token

Token = LLM处理文本的基本单位

内容Token数
1个中文字约1-2个Token
1个英文单词约1-2个Token
1个标点符号约1个Token
1000个汉字约1000-2000个Token

例子：Token是怎么计算的

输入："你好，请帮我写一封邮件" Token化：[你, 好, ，, 请, 帮, 我, 写, 一, 封, 邮, 件] ≈ 12个Token

Token按量计费

LLM不是按"字数"收费，是按"Token"计费。

模型输入价格备注
GPT-4o$2.5/1M Token约0.002元/千Token
Claude 3.5$3/1M Token约0.02元/千Token
MiniMax-M2.7约¥0.01/千Token中文优化

上下文不是"聊天记录"，是LLM的"工作内存"

当LLM回答问题时，它不是在"回忆"——而是在把整个对话历史重新读一遍。

  • 聊天记录 = 给LLM看的材料，不是LLM的"记忆"
  • LLM本身没有状态，每次都是"从零开始"处理上下文
  • 超过上下文窗口的内容，LLM会"忘掉"

四、上下文窗口：LLM的内存限制

上下文窗口 = LLM一次性能处理的Token上限

模型上下文窗口约等于
GPT-4 Turbo128K Tokens约10万汉字
Claude 3200K Tokens约15万汉字
MiniMax-M2.72048K Tokens约150万汉字

例子：上下文越来越长会发生什么

对话轮次 1：消耗 500 Token → OK 对话轮次 10：消耗 5000 Token → OK 对话轮次 100：消耗 50000 Token → 接近上限，前面内容开始被"遗忘"

关键路径：上下文管理的代价

上下文越来越长 → Token消耗越来越快 → 成本越来越高 → 效果可能越来越差

  • 不是塞越多上下文越好，超长上下文会稀释重点
  • 上下文有成本：输入Token + 输出Token = 总消耗
  • 好的Prompt = 在有限上下文里传递最关键的信息

例子：差劲的Prompt vs 好的Prompt

❌ 差劲的Prompt（塞了一堆无关信息）
"我今天早上吃了包子，中午吃了米饭，下午开了会，晚上想点外卖，你知道附近有什么好吃的吗？我喜欢吃辣的，不要太远，最好有优惠"
✅ 好的Prompt（只给关键信息）
"我是外卖用户，喜欢辣味，附近3公里内有什么推荐？要有优惠"

效果几乎一样，但Token消耗少了一半。

核心认知

上下文管理是AI应用的核心竞争力——谁能把上下文用得更好，谁就能做出更强大的AI产品。

❌ 失败模式：大模型幻觉的危害

案例：某电商客服机器人因幻觉给出错误建议，用户投诉激增

背景：某中型电商平台2023年上线了LLM驱动的客服机器人，初期运行良好。半年后，用户开始大量投诉。

用户问："这个保健品我能吃吗？我有高血压和糖尿病"
机器人答："可以的，这个保健品对高血压和糖尿病有辅助治疗作用，建议每天服用两粒。"
实际情况：该保健品含有甘草成分，会与降压药产生相互作用，可能导致血压异常。
后果：一位老年用户听从建议后出现血压骤升紧急就医，平台被投诉到药监局
根因：LLM在训练数据中看到了大量保健品描述，但没有学过医学知识边界，生成了看似专业但实际上错误的建议

幻觉的危害程度量化

场景 幻觉危害程度 后果
一般性问答（闲聊） 低 用户体验下降
产品信息查询 中 购买决策错误
医疗/法律/财务建议 极高 人身伤害/法律风险/财产损失
企业内部决策建议 中-高 商业判断失误

✅ 正确姿势

  • 高风险场景必须人工把关：医疗、法律、财务建议类场景，AI只能做信息整理，不能做决策建议
  • RAG + 知识库约束：让LLM的回答范围限制在已知事实内，减少虚构空间
  • 输出加风险提示：涉及专业领域时，明确告知用户"此信息仅供参考，不构成专业建议"
  • 建立幻觉检测流程：对AI输出的关键事实进行交叉验证，特别是数字、日期、条款类信息
