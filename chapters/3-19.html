课程 3.1：任务拆解

第19章：拆解的方法

STAR法则与拆解层级——把大任务变成可执行的小任务

Page 1 / 2

STAR法则

STAR法则是任务拆解的核心工具，帮助你把模糊的目标变成清晰的执行步骤。

S - Situation 明确背景和目标

我处于什么情况？要达成什么？背景是什么？

问自己：我为什么需要做这件事？最终要什么结果？

T - Task 定义具体任务

我需要做什么？有哪些具体要完成的事项？

问自己：AI能理解这个任务吗？验收标准是什么？

A - Action 描述执行路径

怎么做？谁来做？（人做还是AI做？）先后顺序是什么？

问自己：这个步骤AI能独立完成，还是需要人介入？

R - Result 设定验收标准

怎么算完成了？交付物是什么？质量标准是什么？

问自己：我能判断这个结果是否符合要求吗？

拆解的三个层级

一级拆解：按流程阶段

回答"做什么"——按业务流程划分阶段

  • 准备 → 执行 → 分析 → 输出
  • 或者：调研 → 方案 → 实施 → 复盘

二级拆解：按任务模块

回答"怎么做"——每个阶段有哪些具体模块

  • 准备（问卷设计、受访者招募）
  • 执行（访谈实施、记录整理）
  • 分析（数据清洗、趋势识别）

三级拆解：按操作步骤

回答"具体操作"——每个模块的AI可执行步骤

  • 问卷设计 → AI生成初稿 → 人审核修改 → 定稿
  • 数据分析 → AI清洗 → AI分析 → 人验证结论

📝 示例：把"做一个用户调研"拆解到AI可执行

一级：准备 → 执行 → 分析 → 输出

二级：准备（问卷设计、访谈提纲）
执行（用户访谈、记录整理）
分析（数据清洗、趋势识别）
输出（报告撰写、建议提炼）

三级：问卷设计 → AI生成问卷初稿 → 人审核 → 修改定稿
