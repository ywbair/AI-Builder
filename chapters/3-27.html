课程 3.4：工具链

第27章：搭建个人AI工作流

五步搭建属于你的AI工作流

Page 1 / 2

搭建步骤

步骤一：识别高频任务

列出每天/每周重复做的事情，评估哪些可以AI辅助。

  • 列出日常工作清单
  • 标注每个任务的出现频率
  • 标注每个任务的耗时
  • 优先处理高频且耗时的任务

步骤二：选择合适的AI工具

根据任务类型选择工具，不要用一把锤子解决所有问题。

  • 文本写作 → ChatGPT/Claude
  • 代码相关 → Copilot/Claude Code
  • 信息搜索 → Perplexity/秘塔
  • 知识管理 → Notion/Obsidian

步骤三：设计工作流模板

把流程固化下来，明确输入、处理、输出。

  • 用模板文档描述工作流
  • 明确触发条件和执行步骤
  • 定义审核点和验收标准
  • 预留异常处理方案
Page 2 / 2

继续：测试、优化、沉淀

步骤四：测试和优化

小范围试跑，记录问题并优化，逐步扩大使用范围。

  • 先在非关键任务上测试
  • 记录AI表现和出错类型
  • 调整提示词和工作流
  • 逐步扩大应用场景

步骤五：沉淀为固定流程

形成标准操作程序（SOP），可以分享给团队。

  • 文档化工作流模板
  • 总结常见问题和解决方案
  • 持续迭代优化
  • 考虑团队协作和知识共享

📝 示例：周报AI工作流

步骤一：识别高频任务 → 每周写周报，耗时1小时

步骤二：选择工具 → Claude（擅长长文本分析）

步骤三：设计模板 → 触发(周五5点)→输入(工作记录)→处理(AI整理)→输出(周报草稿)→审核(人修改)

步骤四：测试优化 → 第一版发现AI遗漏某些细节，调整输入格式

步骤五：沉淀 → 形成标准模板，耗时从1小时缩短到20分钟
