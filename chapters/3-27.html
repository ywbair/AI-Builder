课程 3.4：工具链

第27章：搭建个人AI工作流

五步搭建属于你的AI工作流

Page 1 / 2

搭建步骤

步骤一：识别高频任务

列出每天/每周重复做的事情，评估哪些可以AI辅助。

  • 列出日常工作清单
  • 标注每个任务的出现频率
  • 标注每个任务的耗时
  • 优先处理高频且耗时的任务

步骤二：选择合适的AI工具

根据任务类型选择工具，不要用一把锤子解决所有问题。

  • 文本写作 → ChatGPT/Claude
  • 代码相关 → Copilot/Claude Code
  • 信息搜索 → Perplexity/秘塔
  • 知识管理 → Notion/Obsidian

步骤三：设计工作流模板

把流程固化下来，明确输入、处理、输出。

  • 用模板文档描述工作流
  • 明确触发条件和执行步骤
  • 定义审核点和验收标准
  • 预留异常处理方案
Page 2 / 2

继续：测试、优化、沉淀

步骤四：测试和优化

小范围试跑，记录问题并优化，逐步扩大使用范围。

  • 先在非关键任务上测试
  • 记录AI表现和出错类型
  • 调整提示词和工作流
  • 逐步扩大应用场景

步骤五：沉淀为固定流程

形成标准操作程序（SOP），可以分享给团队。

  • 文档化工作流模板
  • 总结常见问题和解决方案
  • 持续迭代优化
  • 考虑团队协作和知识共享

📝 示例：周报AI工作流

步骤一：识别高频任务 → 每周写周报，耗时1小时

步骤二：选择工具 → Claude（擅长长文本分析）

步骤三：设计模板 → 触发(周五5点)→输入(工作记录)→处理(AI整理)→输出(周报草稿)→审核(人修改)

步骤四：测试优化 → 第一版发现AI遗漏某些细节，调整输入格式

步骤五：沉淀 → 形成标准模板，耗时从1小时缩短到20分钟

❌ 失败模式：搭了用不起来

TOP3原因：为什么AI工作流搭了却用不起来

原因1：没有明确触发场景

  • PM学完AI工具后很兴奋，设计了5个"完美"工作流
  • 但平时根本想不起来用，因为没有明确的"什么时候用这个工作流"
  • 结果：工作流躺在笔记里吃灰

原因2：步骤太多记不住

  • PM设计了一个8步的周报工作流
  • 第一次用还顺利，第二次就忘了第5步是什么
  • 第三天想"算了，手写吧"，工作流宣告废弃

原因3：效果不明显放弃

  • PM设计了"竞品分析AI工作流"，但用了2次发现AI生成的分析深度不如自己预期
  • 觉得"这还不如我自己写"，就放弃了
  • 实际上：应该先接受AI初稿的70分水平，再逐步调优，而不是期望一步到位

典型案例：学了一大堆工具，实际工作一个都没用

场景：某PM花了3个月学了ChatGPT、Claude、Copilot、Notion AI、Midjourney等10+工具。

实际工作状态：

  • 每个工具都浅尝辄止，没有一个用精
  • 平时还是用最原始的方式工作
  • 学到的技能因为没有反复练习，一周后就忘了

根因：工具焦虑驱动学习，而不是需求驱动学习

✅ 正确姿势：让工作流用起来的3个技巧

  • 先固化再优化：先用最简单的方式跑通流程（哪怕只有3步），跑通后再加环节优化——而不是一开始设计"完美流程"
  • 设置触发提醒：在日历/待办工具里设置固定提醒，如"每周五下午4点——用AI写周报"
  • 只学用得上的：先识别自己最高频的任务，只学解决那个任务的工具，学到精通再扩展——而不是学一堆工具但都不精
