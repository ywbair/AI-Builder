课程 3.4：工具链

第26章：AI工作流工具

六大类AI工具的类型、用途和选型原则

Page 1 / 2

工具分类

类型一：对话式AI

  • ChatGPT — 通用对话，能力全面
  • Claude — 长文本处理，分析能力强
  • Gemini — Google生态，多模态

核心用途：日常问答、文本生成、知识获取、思路梳理

类型二：编程AI

  • GitHub Copilot — 代码补全、建议
  • Cursor — AI代码编辑器
  • Claude Code — CLI编程助手

核心用途：代码生成、代码审查、Debug、重构

类型三：Agent平台

  • LangChain — Agent开发框架
  • AutoGPT — 自主任务执行
  • CrewAI — 多Agent协作

核心用途：多步骤任务编排、自主执行复杂任务、多Agent协作

类型四：知识库

  • Notion — 文档管理与AI笔记
  • Obsidian — 本地知识图谱
  • 飞书文档 — 团队协作知识库

核心用途：信息整理、文档管理、知识沉淀

Page 2 / 2

更多工具类型

类型五：自动化平台

  • Zapier — 跨工具流程串联
  • Make (Integromat) — 可视化自动化
  • n8n — 开源自动化

核心用途：将不同工具串联起来，实现自动化工作流

类型六：搜索增强

  • Perplexity — AI搜索引擎，带来源
  • 秘塔搜索 — 中文AI搜索
  • Phind — 开发者搜索

核心用途：带来源的深度搜索，获取可靠信息

工具选择三原则

  • 够用就行 — 不追求最多最强，主流工具优先
  • 社区活跃 — 有问题能找到答案，有更新维护
  • 适合自己 — 别人说好的不一定适合你

❌ 失败模式：工具选错导致效率下降

典型案例：用Claude做代码补全而非ChatGPT，因场景不匹配效率反而下降

背景：某PM想用AI辅助写Python数据处理脚本，选择了Claude而不是GitHub Copilot。

对比测试：

工具 使用方式 完成时间 Bug数
Claude 对话式，复制粘贴代码 45分钟 7个
GitHub Copilot IDE内实时补全 15分钟 2个

问题分析：

  • Claude是对话式工具，需要复制粘贴代码片段，上下文切换成本高
  • Copilot是IDE插件，能理解当前代码上下文，实时补全更精准
  • 代码补全场景更适合"上下文感知"的工具

典型案例：用ChatGPT做长文翻译而非专业翻译工具

场景：PM需要将一份产品文档翻译成英文，选择直接用ChatGPT而不是专业翻译工具。

后果：

  • ChatGPT翻译的术语不一致——"用户"一会儿译成"user"，一会儿译成"customer"
  • 产品专有名词翻译错误
  • PM花了2小时人工校对，还不如一开始就用专业翻译工具

✅ 正确姿势：场景匹配优先

  • 代码补全 → GitHub Copilot / Cursor（IDE插件，上下文感知）
  • 长文本分析/写作 → Claude（长上下文，理解能力强）
  • 快速问答/信息检索 → ChatGPT / Perplexity（响应快，来源可追溯）
  • 专业翻译 → DeepL / Google Translate（术语库更专业）
  • 一把钥匙开不了所有锁 — 用对工具才能提效
