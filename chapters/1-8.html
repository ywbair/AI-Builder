第8章：AI Agent时代

Agent架构原理 + RAG + 关键路径

Page 1 / 3

一、什么是AI Agent（不只是聊天机器人）

聊天机器人 = 问答。AI Agent = 完成任务。

维度 聊天机器人 AI Agent
交互模式 一问一答 多步骤执行
目标达成 给出回答 完成具体任务
自主性 被动响应 主动规划执行
工具使用 不能 调用工具、API
典型场景 客服、问答 自动化流程、数据分析

⭐ Agent的核心：规划 + 执行 + 反馈循环

Agent不是一次问答就结束，而是：制定计划 → 执行步骤 → 检查结果 → 调整策略 → 继续执行，直到任务完成。

Page 2 / 3

二、Agent的工作循环（PM必须理解）

Agent 执行循环

1
感知（Perceive）
理解用户意图，分析当前状态和可用工具
↓
2
规划（Plan）
拆解任务，决定下一步做什么（ReAct模式）
↓
3
执行（Act）
调用工具、API或执行计算
↓
4
评估（Eval）
检查执行结果，判断任务是否完成
↻

⭐ Agent为什么容易"跑偏"

Agent每一步都可能犯错——规划错误、执行错误、评估错误。而且这些错误会累积放大。

❌ 典型失败模式：错误累积

第1步错10% → 第2步基于错误信息又错10% → 第3步误差越来越大 → 最终结果完全偏离目标

❌ 典型失败模式：死循环

Agent反复执行同一组操作，无法判断任务已完成或无法完成，陷入无限循环。

⭐ PM的认知：Agent需要"护栏"

设计Agent产品时，必须考虑：最大执行次数、结果验证机制、异常处理预案、人工介入节点。

三、RAG：知识库增强（解决LLM"不知道"的问题）

LLM的知识有截止日期，不知道实时信息，不知道私有数据。RAG就是解决这个问题的。

RAG 工作流程

1
检索
从知识库找出相关内容
→
2
增强
把检索结果加入Prompt
→
3
生成
LLM基于上下文回答

RAG 适用场景

  • 需要回答基于私有文档的问题（合同、制度、文档）
  • 需要引用实时数据（股价、库存、天气）
  • 需要引用特定领域知识（医学、法律、技术）

⭐ 关键路径：上下文管理是AI应用的核心竞争力

ChatBot时期：拼Prompt质量
Agent时期：拼上下文管理能力——谁能更好地组织、检索、注入上下文，谁的产品就更强

🎯 PM的架构认知

  • LLM是引擎——决定推理能力上限
  • 上下文是燃料——决定输出质量
  • Agent是架构——决定能完成什么任务
  • RAG是外挂——解决知识不足的问题

理解这四者的关系，PM就能真正理解AI应用是怎么设计的。
